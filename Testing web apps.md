# Explanation 

Testing means checking if software works right and meets goals.

- Unit (small parts)
- Integrations (how parts work together)
- Functional (if it does what it should)
- UI/UX (look and feel)
- Performance (speed)
- Security
- Accessibility (for disabilities)
- Compatibility (on different devices)

The best ways to write tests with code are:
- [[TDD]]
- [[BDD]]
